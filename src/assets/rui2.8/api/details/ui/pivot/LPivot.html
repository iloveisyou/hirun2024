데이터셋을 기반으로 피벗 기능을 구현한 컴포넌트 트리 그리드 기반으로 구현되어 다중 조건에 대해 엑셀과 유사한 형태로 표현할 수 있다 일반 그리드와 다르게 특정 div에 render함수를 호출할 필요가 없으며 apply 된 곳에 비어있는 그리드와 피벗 다이얼로그를 호출할 수 있는 버튼이 렌더링 된다 피벗 생성 조건은 피벗다이얼로그를 이용하여 설정이 가능하며 화면에 표현되는 값을 alias 처리하기 위해서는 DataSet 선언 시 pivotLabel 값을 넣어주어야 한다. 컴포넌트 높이 설정 시 전체영역(height)과 뷰영역(pivotHeight)은 최소 40px 이상 차이가 나도록 설정한다 성능 문제 브라우져 특성 상 컬럼의 개수가 많아질수록(Dom 수 증가) 성능의 저하가 심하게 일어나기 때문에 컬럼의 개수가 500개가 넘어가면 경고문구를, 2000개가 넘어가게 되면 피벗그리드를 생성 할 수 없다 이는 속성 값으로 변경이 가능하나, 사용자 체감 성능의 하락을 가져올 수 있으니 하나의 피벗 그리드에 많은 컬럼을 표현하지 않도록 주의한다 샘플 경로 : /rui/sample/general/ui/pivot/pivotBasicSample.html

pivot = new Rui.ui.pivot.LPivot({ applyTo: pivotGrid, rawDataSet: dataSet, height: 450, pivotHeight: 400, pivotLabelWidth: 100 });
